# RoadWisdom

**Road Wisdom** คือ Web App Prototype ด้าน Civic Tech ที่ออกแบบมาเพื่อช่วยพัฒนากระบวนการจัดการเรื่องร้องเรียนเกี่ยวกับปัญหาจราจรในกรุงเทพมหานคร โดยระบบนี้ทำหน้าที่เป็นชั้นวิเคราะห์และจัดการข้อมูลเพิ่มเติมจากระบบร้องเรียนที่มีอยู่เดิม เพื่อช่วยเปลี่ยนรายงานปัญหาจราจรที่กระจัดกระจายให้กลายเป็นเคสที่ถูกจัดลำดับความสำคัญ รวมรายงานซ้ำ และติดตามความคืบหน้าได้อย่างชัดเจน

โปรเจกต์นี้พัฒนาขึ้นสำหรับ Hackathon สาย Urban Track โดยมุ่งเน้นเรื่องความปลอดภัยบนท้องถนน การจัดการเรื่องร้องเรียน และการสื่อสารความคืบหน้าระหว่างประชาชนกับเจ้าหน้าที่เมือง

```
Prototype (กำลังพัฒนา) : https://abjust.lovable.app
```
---

## ปัญหาที่ต้องการแก้ไข

กรุงเทพฯ มีช่องทางให้ประชาชนแจ้งปัญหาเมืองอยู่แล้ว แต่รายงานที่เกี่ยวข้องกับการจราจรยังอาจมีข้อจำกัด เช่น

* รายงานจำนวนมากกระจัดกระจาย
* มีรายงานซ้ำจากประชาชนหลายคนในพื้นที่เดียวกัน
* เจ้าหน้าที่จัดลำดับความเร่งด่วนได้ยาก
* ข้อมูลบางส่วนอ่านยากหรือไม่เป็นระบบ
* ประชาชนติดตามความคืบหน้าหลังแจ้งเรื่องได้ไม่ชัดเจน

ปัญหาจราจรบางประเภทไม่ได้กระทบเพียงความสะดวกในการเดินทาง แต่ยังเกี่ยวข้องกับความปลอดภัย เช่น รถจอดผิดกฎหมายใกล้โรงพยาบาล สัญญาณไฟจราจรเสีย ทางเดินเท้าถูกกีดขวาง การใช้ถนนผิดกฎ ป้ายจราจรไม่ชัดเจน หรือถนนน้ำท่วมที่ทำให้เกิดการจราจรติดขัดและอุบัติเหตุ

---

## แนวทางแก้ไข

Prototype นี้ใช้กระบวนการ AI-assisted workflow เพื่อช่วยจัดการรายงานปัญหาจราจรให้เป็นระบบมากขึ้น

```text
ประชาชนแจ้งปัญหา
→ ระบบช่วยสรุปและจัดประเภทเรื่อง
→ ประเมิน Risk Score
→ รวมรายงานซ้ำ
→ แสดงผลบน Officer Dashboard
→ เจ้าหน้าที่อัปเดตสถานะ
→ ประชาชนติดตามผ่าน Timeline
→ ผู้บริหารเมืองดูข้อมูลสะสมผ่าน Analytics
```

เป้าหมายของระบบไม่ใช่การแทนที่ระบบร้องเรียนเดิมของภาครัฐ แต่เป็นการช่วยให้รายงานปัญหาจราจรถูกจัดลำดับ รวมกลุ่ม และติดตามผลได้ง่ายขึ้น

## ฟีเจอร์หลัก

### 1. Citizen Report Form

หน้าสำหรับประชาชนแจ้งปัญหาจราจร โดยสามารถกรอกข้อมูล เช่น

* หัวข้อรายงาน
* ประเภทปัญหา
* ตำแหน่ง
* เขต
* วันและเวลาที่เกิดเหตุ
* รายละเอียดเหตุการณ์
* หลักฐานภาพ/วิดีโอแบบจำลอง
* Checkbox เหตุการณ์อันตรายเร่งด่วน
* Checkbox กระทบกลุ่มเปราะบางหรือรถฉุกเฉิน

ตัวอย่างประเภทปัญหา ได้แก่

* การจอดรถผิดกฎหมาย / สิ่งกีดขวางบนถนน
* การใช้ถนนผิดกฎหรือไม่ปลอดภัย
* สัญญาณไฟจราจรเสียหรือเวลาผิดปกติ
* ป้ายจราจรอันตรายหรือไม่ชัดเจน
* ทางเดินเท้าถูกกีดขวาง
* จุดจราจรติดขัดที่มีความเสี่ยงสูง
* น้ำท่วมถนนที่กระทบการจราจร

---

### 2. AI-assisted Triage Result

หลังจากประชาชนส่งรายงาน ระบบจะแสดงผลการคัดกรองแบบจำลอง ได้แก่

* Case ID
* สรุปเหตุการณ์ที่ระบบสร้าง
* ประเภทปัญหาที่ตรวจพบ
* Risk Score
* Priority Level
* Duplicate Report Grouping
* หน่วยงานที่แนะนำให้รับผิดชอบ

ตัวอย่าง Demo Scenario:

```text
Case: รถจอดผิดกฎหมายกีดขวางทางเข้ารถพยาบาล
Location: ใกล้โรงพยาบาลจุฬาลงกรณ์ ถนนพระราม 4
Risk Score: 92 / Critical
Duplicate Reports: พบรายงานคล้ายกัน 18 รายงานในบริเวณใกล้เคียง
Suggested Unit: Traffic Enforcement Unit
```

---

### 3. Risk Assessment Prioritization

Prototype ใช้ระบบ rule-based scoring เพื่อจำลองการประเมินความเสี่ยงของแต่ละเคส

ปัจจัยที่ใช้ประเมิน เช่น

* ประเภทของปัญหาจราจร
* ความอันตรายเร่งด่วน
* ผลกระทบต่อคนเดินเท้า นักเรียน ผู้สูงอายุ ผู้พิการ หรือรถฉุกเฉิน
* จำนวนรายงานซ้ำ
* ช่วงเวลาเร่งด่วน
* หลักฐานที่แนบมา

ระดับความสำคัญ:

```text
0–39   = Low
40–59  = Medium
60–79  = High
80–100 = Critical
```

---

### 4. Duplicate Report Grouping

ระบบจำลองการรวมรายงานที่เกิดขึ้นในพื้นที่เดียวกันหรือมีลักษณะปัญหาคล้ายกัน เพื่อช่วยลดภาระเจ้าหน้าที่จากการจัดการรายงานซ้ำจำนวนมาก

แทนที่เจ้าหน้าที่ต้องตรวจรายงานหลายรายการจากจุดเดียวกัน ระบบจะรวมเป็นเคสหลักหนึ่งเคส และเจ้าหน้าที่สามารถอัปเดตสถานะให้ผู้แจ้งหลายคนพร้อมกันได้

โดยคำนึงถึงรัศมีเเละหมวดหมู่ของรายงานว่าอยู่ต่ำกว่า150เมตร เเละ มีหมวดหมู่เดียวกันหรือไม่

### 5. Officer Dashboard

Dashboard สำหรับเจ้าหน้าที่จะแสดงเคสที่ถูกจัดลำดับความสำคัญแล้ว พร้อมข้อมูลสำคัญ เช่น

* Case ID
* หัวข้อ
* ตำแหน่ง
* เขต
* ประเภทปัญหา
* Risk Score
* Priority Level
* จำนวนรายงานซ้ำ
* สถานะ
* หน่วยงานที่แนะนำให้รับผิดชอบ

เจ้าหน้าที่สามารถเปิดดูรายละเอียดเคส อัปเดตสถานะ แจ้งผู้รายงาน และทำเครื่องหมายว่าแก้ไขแล้วได้

### 6. Case Detail Page

หน้ารายละเอียดเคสจะแสดงข้อมูลเพิ่มเติม เช่น

* ภาพรวมของเคส
* AI Summary
* Risk Score Breakdown
* รายงานซ้ำที่ถูกรวมไว้ในเคสเดียวกัน
* หลักฐานแบบจำลอง
* หมายเหตุเรื่องการปกป้องข้อมูลส่วนบุคคล
* ปุ่มสำหรับเจ้าหน้าที่ในการดำเนินการ
* การอัปเดตสถานะของเคส

---

### 7. Citizen Timeline Tracking

ประชาชนสามารถติดตามความคืบหน้าของรายงานผ่าน Timeline เช่น

```text
รับเรื่องแล้ว
→ AI สรุปเหตุการณ์แล้ว
→ รวมกับรายงานที่คล้ายกันแล้ว
→ เจ้าหน้าที่กำลังตรวจสอบ
→ ส่งต่อหน่วยงานที่รับผิดชอบ
→ อยู่ระหว่างดำเนินการ
→ แก้ไขเสร็จสิ้น
→ แจ้งผู้รายงานแล้ว
```

ฟีเจอร์นี้ช่วยให้ประชาชนรู้สึกว่ารายงานของตนไม่หายไปจากระบบ และเพิ่มความโปร่งใสในการจัดการปัญหาเมือง

### 8. Analytics / Manager Dashboard

หน้า Analytics ใช้ mock data เพื่อแสดงให้เห็นว่าข้อมูลสะสมจากรายงานสามารถนำไปใช้วางแผนเมืองในระยะยาวได้

ตัวอย่างข้อมูลที่แสดง:

* ประเภทปัญหาจราจรที่พบบ่อย
* เขตที่มีรายงานสูง
* จุดที่เกิดปัญหาซ้ำ
* เวลาเฉลี่ยในการตอบสนอง
* จำนวนเคสที่แก้ไขแล้วในสัปดาห์นี้

## จริยธรรม ความเป็นส่วนตัว และความปลอดภัย

เนื่องจากระบบเกี่ยวข้องกับภาพ วิดีโอ ตำแหน่ง และเรื่องร้องเรียนด้านจราจร โปรเจกต์นี้จึงยึดหลักดังนี้

### 1. Privacy by Design

ออกแบบระบบโดยคำนึงถึงความเป็นส่วนตัวตั้งแต่แรก

### 2. Data Minimization

เก็บเฉพาะข้อมูลที่จำเป็นต่อการตรวจสอบและแก้ไขปัญหาเท่านั้น

### 3. การปกป้องใบหน้าและป้ายทะเบียน

หากพัฒนาต่อเป็นระบบจริง ภาพและวิดีโอควรถูกเบลอใบหน้าและป้ายทะเบียนก่อนนำไปตรวจสอบหรือแสดงผล

### 4. Human-in-the-loop

AI ทำหน้าที่ช่วยสรุป จัดประเภท และประเมินความเสี่ยง แต่ไม่ควรตัดสินใจแทนมนุษย์

```text
AI แนะนำ แต่มนุษย์ตัดสินใจ
```

### 5. Explainable Risk Score

ระบบควรแสดงเหตุผลของคะแนนความเสี่ยง เช่น ระดับความอันตราย จำนวนรายงานซ้ำ ผลกระทบต่อรถฉุกเฉิน หรือช่วงเวลาเร่งด่วน

### 6. ไม่ใช้ข้อมูลส่วนบุคคลจริงใน Prototype

Prototype นี้ใช้ mock data เท่านั้น และไม่เก็บข้อมูลส่วนบุคคลจริงของประชาชน
